# Blueprint 2: A Memory

## Purpose

A memory is the smallest meaningful piece of a person's life.

It is not a file, document, database record, or entry. It is a moment, experience, reflection, lesson, or feeling that someone chooses to preserve.

## Emotional Objective

Creating a memory should feel like speaking beside a trusted companion around a campfire.

## Behaviour

- A memory begins with an invitation, not a blank form.
- The user may type, speak, pause, add media, or continue later.
- No Save button exists.
- Silence is treated as part of remembering.
- If the user pauses, Me-Wall may gently display: “Take your time.”
- Me-Wall asks gentle questions only after listening.
- A memory never ends as “complete.” It remains open and may grow.

## Gentle Questions

- Who else was part of this memory?
- Where did this happen?
- How did this change you?
- What do you remember most clearly?
- Is there anything else that comes to mind?

## Design Law

Me-Wall preserves memories as they are remembered, not as the computer wishes they had been remembered.
